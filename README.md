# 只見川 川霧 観測・予測プログラム（weather_visualizer.py）

只見川流域の気象データ（気象庁ダウンロード形式のExcel）を読み込み、

1. **気温・湿度・風速・降水量などの気象データ**と、
2. **32地点の川霧・層雲・霧雨などの現象コード**

を月ごとにグラフ化し、さらに **scikit-learn（RandomForestClassifier）で地点ごとに霧の発生を学習** して、
**Open-Meteo API から取得した今後16日間の気象予報**をもとに、地点ごとの霧予測グラフまで自動生成するツールです。

---

## できること

- 📊 月ごとの気象データ×現象コードのグラフ生成（32地点ぶんの現象レーンつき）
- 🧹 現象コード列の中に「/」がほぼ皆無で不自然に均等分布しているブロック（テスト用ダミーデータの疑いがある期間）を自動検出し、学習データから除外
- 🤖 32地点それぞれについて独立したRandomForestClassifierを学習（多クラス分類：「/」+コード1〜10）
- 🎛️ `RandomizedSearchCV`による地点ごとのハイパーパラメータ自動調整（サンプルが少ない地点は既定値に自動フォールバック）
- 🔮 Open-Meteoの無料予報APIから今後16日分の気象予報を取得し、学習済みモデルで地点ごとに現象コードを予測
- 🖼️ 予測結果を、実測データと同じ配色・同じ形式のグラフとして地点ごとに個別出力（予測ゾーンは背景の網掛けと境界線で明示、色自体は変えない）
- 📈 全地点の予測をまとめた「日別霧予測サマリー」グラフ（32枚を1枚ずつ見なくても全体感がつかめる）

---

## 必要な環境・ライブラリ

```bash
pip install pandas numpy matplotlib openpyxl scikit-learn requests
```

インターネット接続が必要です（Open-Meteoへの予報取得のため）。日本語フォントが無い環境（Colab等）では、
初回実行時に `fonts-noto-cjk` の自動インストールを試みます。

## 使い方

```bash
python3 weather_visualizer.py 入力ファイル.xlsx [出力フォルダ] [--no-forecast]
```

- 出力フォルダを省略した場合は `./output_graphs` に保存されます。
- `--no-forecast` を付けると、月別グラフ（①②）のみ生成し、霧予測（④⑤）はスキップします。
- Jupyter / Google Colab のセルにそのまま貼り付けて実行することもできます。
  - 入力ファイルが見つからない場合、Colabなら自動でアップロード画面が表示されます。

## 入力Excelのフォーマット

気象庁からダウンロードした形式を想定しています。

| 列 | 内容 |
|---|---|
| A列 | 年月日時（1時間ごと） |
| B列 | 気温(℃) |
| E列 | 降水量(mm) |
| H列 | 風速(m/s) |
| S列 | 露点温度(℃) |
| V列 | 相対湿度(％) |
| AC列〜BH列 | 32地点ぶんの現象コード（1地点＝1列） |
| 3行目 | AC〜BH列それぞれの観測地点名 |
| 4行目付近 | A列に「年月日時」と書かれたヘッダー行（自動検出） |

### 現象コードの意味

| コード | 意味 | 分類 |
|---|---|---|
| `/` | 現象なし | - |
| （空欄） | まだ観測結果が入力されていない | - |
| 1 | 薄い川霧 | 霧 |
| 2 | 川霧 | 霧 |
| 3 | 濃い川霧 | 霧 |
| 4 | 薄い全体霧 | 霧 |
| 5 | 全体霧 | 霧 |
| 6 | 全体濃い霧 | 霧 |
| 7 | 薄い層雲 | 雲 |
| 8 | 濃い層雲 | 雲 |
| 9 | 霧雨 | 雨 |
| 10 | 雨 | 雨 |

`/`（現象なし）と空欄（未入力）はExcel上は見た目が似ていますが、プログラム内部では区別しています
（`encode_phenomena_cell()`が空文字列は欠測、`/`は現象なし=コード0として扱います）。

## 出力されるグラフ

### ①②：気象データ × 現象コード（月別・全地点まとめ）

既存の可視化部分（変更していません）。月ごとに以下の2種類を生成します。

- `{地点名}_①気温・湿度・露点×現象コード_YYYY-MM.png`
- `{地点名}_②風速・降水量×現象コード_YYYY-MM.png`

上段が気温・湿度・露点温度・風速・降水量の時系列、下段が32地点ぶんの現象コードを
色分けした帯（レーン）で表示します。横軸の日付は `年/月/日` 形式です。

### ④：地点ごとの16日間予測グラフ（追加機能）

学習済みモデルで予測できた地点それぞれについて、以下のファイルを個別に生成します。

- `{地点名}_④{観測地点名}_16日間予測_YYYYMMDD.png`

- **上段**：直近5日間の実測気温・露点・湿度（実線）＋ 今後16日間の予報（同色・点線）
- **下段**：その地点の現象コード。実測・予測とも既存と同じ配色で表示し、
  予測ゾーンであることは背景のうっすらしたグレーと境界線（縦の点線）だけで示します
- 目盛りは「年/月/日」で表示し、実測と予測がどちらの年のデータかひと目でわかるようにしています

### ⑤：日別霧予測サマリー（追加機能）

学習済みモデルがある地点をまとめ、日ごとに「霧（コード1〜6）が予測された地点数」を棒グラフにします。

- `{地点名}_⑤日別霧予測サマリー_YYYYMMDD.png`

32枚の個別予測グラフを1枚ずつ見なくても、全体の傾向を把握できます。

## 霧予測の仕組み

1. **目的変数**：地点ごとに、その時刻の現象コード（0=`/`〜10）をそのまま多クラス分類のラベルとして使います
   （霧か否かの二値化はしていません＝元データと同じ粒度で予測します）。
2. **学習データ**：**実際に観測結果が入力されている行だけ**を使います。空欄（未観測）の行は自動的に除外されます
   （`train_location_models()`内の`dropna(subset=[col])`）。
   さらに、`find_dummy_blocks()`により、一定期間まとまって「/」がほぼ皆無（現象コードが不自然に均等分布）
   なブロックを検出し、テスト用ダミーデータの疑いがあるものとして学習データから自動的に除外します。
3. **特徴量**：気温・降水量・風速・相対湿度・露点温度に加えて、
   - 気温と露点温度の差（気温露点差）
   - 時刻をsin/cosに変換したもの（0時と23時が近いことをモデルに教えるため）
   - 月
4. **モデル**：`RandomForestClassifier`を**32地点それぞれに個別に学習**します。地点間でデータを混ぜることはしません。
   - 評価用の train/test 分割は**時系列順を保ち**、後ろ20%をテストに使います（未来データが学習に混ざらないようにするため）
   - `RandomizedSearchCV`（`TimeSeriesSplit`）で`n_estimators`・`max_depth`・`min_samples_leaf`・`max_features`を地点ごとに自動探索します
   - 交差検証の分割数が足りない場合は探索をスキップし、既定値（`n_estimators=500, max_depth=14, min_samples_leaf=2`）にフォールバックします
   - データ件数が少なすぎる地点（200件未満）や、現象の種類が1種類しかない地点は自動的にスキップされます
   - 学習後、地点ごとに正解率・F1スコア・**霧F1**（コード1〜6に限定した macro F1）を一覧表示します
     （「学習完了(調整済)」＝ハイパーパラメータ探索成功、「学習完了(既定値)」＝フォールバック）
5. **予測**：Open-Meteo（無料・APIキー不要）から**実行した日（今日）以降**の気象予報を取得し、
   学習済みモデルにそのまま入力して地点ごとの現象コードを予測します。

## 既知の制限・注意点

- **季節性の偏り**：只見川の川霧は主に6〜9月の早朝・夕方に発生しやすいため、その季節を含まないデータ期間では
  霧サンプルがほぼゼロになり、学習できない（またはあまり意味のある予測にならない）地点が出てきます。
- **観測記録と予報のあいだの空白**：Excelの現象コード列は、気象データ本体（気温など）より入力が遅れている
  ことがあります。その場合、「最後に観測結果が入力されている日」と「予報開始日（今日）」の間に数日〜数百日の
  空白が生じることがあります（3日以上の空白がある場合はコンソールに警告を表示します）。この空白そのものは
  埋められません（過去の実測値がないため）。
- **座標**：予報取得地点の緯度経度はコード上部の`FORECAST_LAT` / `FORECAST_LON`にデフォルト値
  （会津若松付近）を設定しています。実際の観測地点の正確な座標が分かる場合は書き換えると精度が上がります。
- **インターネット接続が必須**：予測グラフの生成にはOpen-Meteo APIへのアクセスが必要です。ネットワークに
  出られない環境（本開発時のサンドボックス環境など）では、学習・評価までは動きますが予報取得の部分でエラーに
  なり、その旨のメッセージが表示されます。
- **列の位置・地点数はハードコード**：`COL_TEMP`, `PHENOMENA_RANGE`などの列位置は現在のExcel形式に
  合わせて固定値になっています。フォーマットが変わった場合はコード上部の定数を書き換える必要があります。

## 主要な関数（コードを読むときの目印）

| 関数 | 役割 |
|---|---|
| `load_weather_data()` | Excelを読み込み、気象データ(main_df)・現象コード(phenom_df)・地点名一覧を作る |
| `encode_phenomena_cell()` | セルの値（`/`・数字・空欄）を内部コード（0〜10・NaN）に変換 |
| `find_dummy_blocks()` | 「/」がほぼ皆無なまとまった期間を検出し、ダミーデータの疑いがあるブロックとして報告 |
| `plot_temp_humid_dew()` / `plot_wind_precip()` | ①②のグラフを1ヶ月ぶん描画 |
| `plot_combo_by_month()` | 月ごとに上記2つのグラフをまとめて生成 |
| `train_location_models()` | 32地点それぞれのRandomForestClassifierを学習 |
| `fetch_forecast_range()` | Open-Meteoから今後16日分の気象予報を取得 |
| `build_location_forecast_codes()` | 予報データに学習済みモデルを適用し、地点ごとの現象コードを予測 |
| `plot_single_location_forecast()` | ④の地点別予測グラフを1地点ぶん描画 |
| `plot_all_location_summary()` | ⑤の全地点サマリーグラフ（日別・霧予測地点数の棒グラフ）を描画 |
| `run_fog_prediction_addon()` | 上記を一通り実行するエントリーポイント（`main()`の最後で呼ばれる） |
